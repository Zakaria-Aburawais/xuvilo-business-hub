FEATURE 1 — INVOICE TRACKING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Allow users to track the status of sent invoices
without requiring an account.

1. SHAREABLE INVOICE LINK
   After generating an invoice, show a 
   "Share & Track" button next to Download PDF.
   
   When clicked:
   - Generate a unique 8-character ID for the invoice
     e.g. "INV-X7K2M9P4"
   - Store the invoice data in localStorage with this ID
   - Create a shareable URL:
     /invoice/track/[ID]
   - Show the URL in a modal with:
     - Copy link button
     - WhatsApp share button
     - "Your client can view and confirm this invoice 
       at the link above"

2. CLIENT VIEW PAGE
   Create route: /invoice/track/:id
   
   This page shows the invoice to the client:
   - Full invoice preview (read-only)
   - Three action buttons at the top:
     ✅ "Confirm Receipt / تأكيد الاستلام"
     💬 "Send Message / إرسال رسالة"  
     ⬇️ "Download PDF / تحميل PDF"
   
   When client clicks "Confirm Receipt":
   - Show a confirmation modal:
     "Confirm you have received this invoice?"
     [Confirm] [Cancel]
   - On confirm: update invoice status to "Seen ✅"
   - Store status update in localStorage
   - Show thank you message:
     "Thank you! The sender has been notified."

3. INVOICE STATUS TRACKING
   On the sender's invoice form, add a 
   "My Invoices" section or link that shows:
   
   A simple list of recent invoices stored locally:
   
   ┌─────────────────────────────────────────┐
   │ #INV-001  |  Client Name  |  $1,250  │
   │ Status: 👁 Seen ✅  |  2026-04-21      │
   ├─────────────────────────────────────────┤
   │ #INV-002  |  Client Name  |  $800    │
   │ Status: 📤 Sent  |  2026-04-20         │
   └─────────────────────────────────────────┘
   
   Status options:
   - 📤 Sent (default when link is shared)
   - 👁 Seen (client opened the link)
   - ✅ Confirmed (client clicked confirm)
   - 💰 Paid (sender manually marks as paid)
   - ❌ Overdue (past due date, auto-detected)
   
   Add a "Mark as Paid" button on each invoice row.
   
   Store all data in localStorage under key:
   "businesseshub_invoices"

4. TRACKER PAGE
   The existing /tracker route should become
   this invoice tracking dashboard.
   
   Show all locally stored invoices in a clean table:
   - Invoice number
   - Client name
   - Amount + currency
   - Issue date / Due date
   - Status badge (color coded)
   - Actions: View | Share Link | Mark Paid | Delete
   
   Add filter tabs at top:
   All | Sent | Seen | Confirmed | Paid | Overdue
   
   Add summary cards at top:
   ┌──────────┐ ┌──────────┐ ┌──────────┐
   │ Total    │ │  Paid    │ │ Pending  │
   │ $12,450  │ │  $8,200  │ │  $4,250  │
   └──────────┘ └──────────┘ └──────────┘

FEATURE 2 — NO SIGN-UP FREE TRIAL FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Remove ALL friction from first use. Users should
be able to create and download a complete invoice
without any account, email, or sign-up.

1. REMOVE SIGN-UP GATES
   Audit every point where the app asks for 
   sign-up or login before allowing action.
   
   The following must work WITHOUT any account:
   - Creating an invoice ✓ (already works)
   - Downloading as PDF ✓ (already works)
   - Changing templates
   - Using calculators
   - Browsing all templates
   - Using stamp maker
   - Using temp email tool
   - Sharing invoice tracking link
   
   If any of these currently require login,
   remove that requirement immediately.

2. SOFT SIGN-UP PROMPT (not a gate)
   After a user downloads their SECOND invoice:
   Show a non-blocking banner at the top:
   
   "💾 Save your invoices across devices — 
   Create a free account to never lose your work.
   [Create Free Account]  [Maybe Later]"
   
   - "Maybe Later" dismisses for the session
   - Never show this before the second download
   - Never block the download to show this
   - Store dismissal in localStorage

3. GUEST MODE INDICATOR
   In the top nav, where Login/Sign Up buttons are,
   add a subtle indicator for guest users:
   
   Show a small pill: "👤 Guest Mode"
   On hover/click show tooltip:
     "You're using Businesses Hub as a guest.
      Your invoices are saved locally on this device.
      Create a free account to sync across devices."
   
   This replaces feeling like a stranger with
   feeling like a valued guest.

4. LOCAL STORAGE AUTO-SAVE
   Every invoice the user creates should be 
   auto-saved to localStorage immediately:
   - Save on every form change (debounced 2s)
   - On page load, restore last invoice state
   - Show subtle "Auto-saved ✓" indicator
   - Allow user to see "Recent Invoices" 
     without signing in
