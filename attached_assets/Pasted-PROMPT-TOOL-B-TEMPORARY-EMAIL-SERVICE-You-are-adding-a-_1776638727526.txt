STEP 1 — BACKEND API PROXY (Next.js API Routes)
Create proxy routes at /api/tempmail/ to avoid CORS issues with the mail.tm API:
/api/tempmail/domains        → proxies GET    https://api.mail.tm/domains
/api/tempmail/accounts       → proxies POST   https://api.mail.tm/accounts
/api/tempmail/token          → proxies POST   https://api.mail.tm/token
/api/tempmail/messages       → proxies GET    https://api.mail.tm/messages  (requires Authorization header)
/api/tempmail/message/[id]   → proxies GET    https://api.mail.tm/messages/{id}
/api/tempmail/delete/[id]    → proxies DELETE https://api.mail.tm/messages/{id}
Add basic rate limiting: max 30 requests per minute per IP using an in-memory counter in the API route.
