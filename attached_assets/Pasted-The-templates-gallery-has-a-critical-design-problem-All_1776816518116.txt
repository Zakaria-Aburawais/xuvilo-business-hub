The templates gallery has a critical design problem.
All templates within each category look identical 
— same layout, same structure, only colors differ.
110 templates is misleading if they're all the same 
design with different color schemes.

Each template must have a VISUALLY DISTINCT layout.
Not just different colors — different structure,
different header styles, different table designs,
different typography, different information hierarchy.

Build these 15 structurally different invoice 
layouts. Each must look completely different

- White background, company name top-left
- Invoice title top-right in large text

- Bold grand total box bottom-right

LAYOUT 3 — Left Sidebar
- Colored left sidebar (20% width)
- Company info + logo in sidebar

- Very modern, magazine-style look

LAYOUT 4 — Top Color Band
- Thin colored band at very top (like a ribbon)
- Large company logo centered below band
- Two-column layout: billed-from left,

- Minimal table, no borders just lines
- Totals in colored box bottom-right

LAYOUT 5 — Centered Minimal
- Everything centered
- Large invoice number as hero element
- Company and client info in two centered blocks
- Table full width, borderless
- Footer with payment terms centered

LAYOUT 6 — Bold Corporate
- Large colored header 40% of page height
- Company name in large white text in header
- Document type "INVOICE" as watermark 
  behind header

- Heavy typography throughout

LAYOUT 7 — Arabic RTL Classic
- Full RTL layout, right-to-left
- Arabic header with company name right
- Invoice number right, date right
- Table headers in Arabic
- Totals on left side (RTL standard)

LAYOUT 8 — Arabic Modern
- RTL layout with colored right sidebar
- Arabic typography throughout
- Bilingual labels (Arabic primary, 
  English secondary smaller)
- Modern geometric Arabic design

LAYOUT 9 — Bilingual Split
- Header split 50/50: Arabic right, English left
- Table with bilingual column headers
- All labels shown in both languages
- Suitable for international business

LAYOUT 10 — Freelancer Simple
- No table — line items as simple rows
- Large payment amount as hero element
- Minimal fields: just what's needed
- "Thank you for your business" 
  as prominent footer element
- Warm, personal feel

LAYOUT 11 — Government/Formal
- Official stamp placeholder top-right
- Sequential reference numbers
- Formal border around entire document
- All caps headings
- Signature and stamp area at bottom
- Suitable for Libya/MENA government contracts

LAYOUT 12 — Construction/Engineering
- Extra columns: Materials, Labor, Equipment
- Project name and site address fields
- Phase/milestone breakdown rows
- Professional dark green or steel blue accent
- Suitable for contractors

LAYOUT 13 — Consulting/Services
- Hourly rate column
- Hours worked column
- Project name field prominently
- Professional grey + blue palette
- Suitable for consultants and agencies

LAYOUT 14 — Elegant Premium
- Serif typography (Playfair Display or similar)
- Gold or warm brown accent color
- Decorative thin border lines
- Logo centered at top
- Suitable for luxury services, legal, finance

LAYOUT 15 — Compact Print
- Designed to fit on half A4
- Very dense, small font
- All information visible at once
- No wasted whitespace
- Suitable for retail/point of sale

IMPLEMENTATION REQUIREMENTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

For each template:
1. Build as a real React component with 
   completely different JSX structure
2. Do NOT just change CSS variables/colors —
   the HTML structure itself must be different
3. All templates must accept the same props
   (businessInfo, clientInfo, items, totals)
   so they're interchangeable
4. All templates must render correctly in PDF
5. All templates must support RTL when Arabic
   language is selected
6. Thumbnails in the gallery must accurately
   reflect the actual template layout —
   not all show the same structure

GALLERY DISPLAY FIX
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The template gallery thumbnails must show the 
ACTUAL layout of each template.

Currently all thumbnails look identical.
Fix by rendering a mini live preview of each 
template at 20% scale using CSS transform:

<div style={{ 
  transform: 'scale(0.2)', 
  transformOrigin: 'top left',
  width: '500%',
  height: '500%',
  pointerEvents: 'none'
}}>
  <TemplateComponent {...sampleData} />
</div>

Wrap in a fixed-size container with overflow:hidden
so it appears as a proper thumbnail.

This way users see the REAL template layout
not a generic placeholder.
