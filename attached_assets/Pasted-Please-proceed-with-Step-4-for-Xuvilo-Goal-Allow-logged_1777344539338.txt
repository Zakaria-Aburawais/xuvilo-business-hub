Please proceed with Step 4 for Xuvilo.

Goal:
Allow logged-in users to save, view, edit, duplicate, delete, print, and export their invoices, quotations, and receipts from their SaaS workspace.

Important instructions:
- Do not break authentication, dashboard, role support, rate limiting, company profile, clients, auto-fill, AI Writer, calculators, templates, pricing, blog, or country pages.
- Do not integrate Stripe or live payments yet.
- Do not rebuild the existing invoice, quotation, or receipt tools from scratch.
- Improve existing tools incrementally.
- Keep anonymous/public users able to use the document tools manually if they currently can.
- All saved documents must be scoped to the logged-in user.
- Do not expose one user's documents to another user.

Part 1: Database models
Create or verify database tables for saved documents.

Option A: one unified documents table:
- id
- user_id
- document_type: invoice | quotation | receipt
- document_number
- title
- client_id nullable
- client_snapshot JSON
- company_snapshot JSON
- items JSON

- currency
- language
- status
- notes
- terms
- issue_date
- due_date nullable
- valid_until nullable
- paid_date nullable
- created_at
- updated_at

Option B: separate tables:
- invoices
- quotations
- receipts

Choose the option that best fits the current codebase, but avoid unnecessary complexity.

Important:
Use snapshots for company/client data so old saved documents do not change unexpectedly if the user later edits company profile or client details.

Part 2: Backend API routes
Add authenticated API routes for saved documents.

Recommended routes:
- GET /api/documents
- POST /api/documents
- GET /api/documents/:id
- PUT /api/documents/:id
- DELETE /api/documents/:id
- POST /api/documents/:id/duplicate

Support filters:
- type=invoice|quotation|receipt
- status
- client_id
- search
- date_from
- date_to

Requirements:
- All routes require authentication.
- All records must be scoped to the logged-in user.
- Validate document_type.
- Validate required fields.
- Return consistent JSON envelopes.
- Do not allow users to access, edit, duplicate, or delete another user's documents.

Success response:
{
  "success": true,
  "data": { ... }
}

List response:
{
  "success": true,
  "data": [...],
  "pagination": { ... }
}

Error response:
{
  "success": false,
  "error": "Invalid request",
  "message": "Please provide a document type and document number."
}
