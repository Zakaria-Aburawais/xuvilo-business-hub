Part 2: Backend API routes
Add authenticated API routes:

Company profile:
- GET /api/company-profile
- POST /api/company-profile
- PUT /api/company-profile

Clients:
- GET /api/clients
- POST /api/clients
- GET /api/clients/:id
- PUT /api/clients/:id
- DELETE /api/clients/:id

Requirements:
- All routes require authentication.
- All records must be scoped to the logged-in user.
- Return consistent JSON envelopes.
- Validate inputs.
- Do not allow users to access, edit, or delete another user's records.

Example success response:

{
  "success": true,
  "data": { ... }
}

Example validation error:

{
  "success": false,
  "error": "Invalid request",
